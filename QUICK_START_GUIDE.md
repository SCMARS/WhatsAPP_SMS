```
Добавь параметр "language" в API request

language: "ru" → Русское сообщение от Марии 🇷🇺
language: "uz" → Узбекское сообщение от Nozima 🇺🇿
```

## 📝 Примеры (Copy-Paste)

### 1️⃣ Русское сообщение одному контакту

```bash
curl -X POST http://localhost:8000/api/telegram/send \
  -H "X-API-Key: YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "phone": "+998901234567",
    "lead_name": "Владимир",
    "language": "ru"
  }'
```

**Результат:** Сообщение на русском в 3 части ✅

---

```bash

  -H "X-API-Key: YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "campaign_external_id": "uzbekistan",
    "leads": [
      {"phone": "+998901111111", "lead_name": "Владимир", "language": "ru"},
      {"phone": "+998902222222", "lead_name": "Alisher", "language": "uz"},

      {"phone": "+998904444444", "lead_name": "Sardor", "language": "uz"}

  }'
```

**Результат:** 
- 2 контакта получат русское сообщение (Владимир, Сергей) 🇷🇺
- 2 контакта получат узбекское сообщение (Alisher, Sardor) 🇺🇿

## ❓ FAQ

### Q: Что если не передать language?
**A:** Система использует язык по стране
- +998 (Узбекистан) → узбекский (uz)
- +351 (Португалия) → португальский (pt)
- +54 (Аргентина) → испанский (es)

### Q: Каждый контакт получит свою ссылку?
**A:** ДА! Система автоматически:
1. Берет следующую свободную ссылку из пула
2. Подставляет её в сообщение
3. Отправляет контакту

### Q: Сколько частей в сообщении?
**A:** Всегда 3 части:
1. Приветствие (82-99 символов)
2. Основное предложение + ссылка (166-180 символов)
3. Призыв к действию (69-73 символов)

Отправляются с интервалом 3-6 секунд между частями.

### Q: Что если закончатся ссылки?
**A:** Ошибка 503:
```json
{
  "status": "error",
  "detail": "Link pool exhausted for country=UZ"
}
```
Загрузите новые ссылки через `/api/links/load`

### Q: Язык русский, но в узбекском контексте?
**A:** Все сообщения содержат "8.000.000 UZS" и "Sweet Bonanza" - совместимо для обоих языков.
