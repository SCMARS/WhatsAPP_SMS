## 🔍 Как это работает

### Шаг 1: Определение страны по номеру
```
Номер телефона → Определение страны → Язык по умолчанию
+998901234567 → Узбекистан (UZ) → узбекский (uz)
```

### Шаг 2: Применение параметра language (если передан)
```
Если language='ru' передан → Используется русский
Если language='uz' передан → Используется узбекский  
Если не передан → Используется язык по стране
```

### Шаг 3: Выбор шаблона сообщения
```
language='ru' → Шаблон от Марии (русский)
language='uz' → Шаблон от Nozima (узбекский)
```

### Шаг 4: Подстановка уникальной ссылки
```
Система берет следующую доступную ссылку из пула
Подставляет её в сообщение Part 2 (предложение)
Каждый контакт получает свою уникальную ссылку
```

### Шаг 5: Отправка 3 частей сообщения
```
Part 1 → Приветствие (случайная вариация)
Part 2 → Основное предложение с уникальной ссылкой
Part 3 → Призыв к действию и инструкция активации
```

## 📡 Параметры API

### Обязательные параметры:
| Параметр | Тип | Описание | Пример |
|----------|-----|---------|---------|
| `phone` | string | Номер телефона | `+998901234567` |

### Опциональные параметры:
| Параметр | Тип | Описание | Значения | Приоритет |
|----------|-----|---------|----------|-----------|
| `language` | string | **Язык сообщения** | `"ru"` или `"uz"` | 🔴 **НАИВЫСШИЙ** |
| `lead_name` | string | Имя контакта | любая строка | низкий |
| `lead_id` | string | ID контакта | любая строка | низкий |
| `campaign_external_id` | string | ID кампании | любая строка | низкий |
| `batch_index` | int | Индекс для батча | 0, 1, 2... | низкий |

### Приоритет выбора языка:
```
1️⃣  language="ru" или language="uz" → ИСПОЛЬЗУЕТСЯ ЯВНО УКАЗАННЫЙ
2️⃣  Если не указан → ИСПОЛЬЗУЕТСЯ ПО СТРАНЕ
3️⃣  +998 → узбекский (uz)
    +351 → португальский (pt)
    +54  → испанский (es)
```

## 📨 Примеры использования

### ✅ ВАРИАНТ 1: Отправить Русское сообщение

**Single Send (одному контакту):**
```bash

  -H "X-API-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "phone": "+998901234567",
    "lead_name": "Владимир",
    "language": "ru"
  }'
```

## ✨ Особенности

### 1️⃣ **Уникальные ссылки для каждого контакта**
```
Lead 1 → link_id=100
Lead 2 → link_id=101
Lead 3 → link_id=102
...
Lead N → link_id=N+99
```
Каждая ссылка уникальна и отслеживается отдельно.

### 2️⃣ **Вариативность сообщений (Анти-фильтрация)**
```
3 вариации приветствия
3 вариации предложения
3 вариации призыва к действию

= 27 потенциальных комбинаций
→ Избегает фильтрации по идентичному контенту
```

### 3️⃣ **3-частное сообщение**
```
Part 1 (82-99 символов) → Приветствие
Part 2 (166-180 символов) → Предложение + ССЫЛКА
Part 3 (69-73 символов) → Инструкция

Отправляются с интервалом 3-6 секунд
→ Избегает автоматического блокирования
```

### 4️⃣ **Поддержка разных языков в одной рассылке**
```
Одна команда → разные языки одновременно
Без необходимости разбивать на отдельные кампании
```

### 5️⃣ **Fallback на язык по стране**
```
Если language не передан:
  +998 → узбекский (uz)
  +351 → португальский (pt)
  +54  → испанский (es)
```

## 🔐 Встроенные проверки

### Проверка 1: Валидация страны
```
✅ Поддерживаемые: PT, AR, UZ
❌ Остальные: ошибка 422
```

### Проверка 2: Проверка пула ссылок
```
✅ Есть свободные ссылки → отправка
❌ Пул пуст → ошибка 503
```

### Проверка 3: Блэклист
```
✅ Контакта нет в блэклисте → отправка
❌ Контакт в блэклисте → статус "blacklisted"
```

### Проверка 4: Валидация языка
```
✅ language='ru' → русский
✅ language='uz' → узбекский
✅ language не указан → по стране
❌ language='xx' → неизвестный язык (игнорируется, используется по стране)
```
